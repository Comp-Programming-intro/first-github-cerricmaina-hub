# What I Learned About GitHub

In this introductory exercise, I learned the core concepts of version control and how the GitHub flow operates:

* **Git vs. GitHub:** Git is a local tool used to track changes and take snapshots of my code, while GitHub is the cloud platform where those repositories are stored and shared.
* **Repositories & Cloning:** A repository is like a digital project folder. Cloning allows me to download a copy of that remote folder from the cloud to work on it locally on my machine.
* **Committing & Pushing:** Committing creates a safe local checkpoint of my work with a descriptive message. Pushing uploads those local checkpoints so my instructor can see them.
* **Collaboration Tools:** Features like Branches, Pull Requests, and Issues allow developers to safely experiment with code and collaborate smoothly without breaking the main project.
